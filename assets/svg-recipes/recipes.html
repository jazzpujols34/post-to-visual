SVG recipe library

Vetted, copy-paste SVG patterns for post-to-visual pages. Every recipe is themeable (CSS-var fills, light + dark), obeys the layout rules (unique marker ids, two-line labels, computed viewBox, min-width to scroll on mobile), and uses --*-solid tokens so white-text boxes never break in dark mode.

How to use this file. It is both a gallery and the source. Pick a recipe with the guide below, then copy its SVG (button on each card) or just read this .html directly. Adapt freely — this is a recipe set, not a template engine. The model still composes per post. Toggle the theme (top-right) to confirm a diagram reads in both modes before shipping.

Tweak: this one is HTML (not SVG) — fastest path for stats. Swap .big color per tile via inline color:var(--teal). Grid auto-fits; keep 2–5 tiles. Use SVG bar instead when the numbers should be compared by length.
barcompare discrete quantities by length
Tweak: bar width = value% × usable width (here 480px = 100%). Put the value inside the bar in #fff only when the bar is long enough; otherwise place it just past the end in var(--text-primary). Bars use --*-solid so they stay saturated in dark mode. Row pitch 42px.
flowa linear sequence of steps / a pipeline
Tweak: N boxes at even x-pitch (here 234px), each 186×58, two-line (label + sub). Arrows span the 46px gaps. For 3 steps widen boxes; for 5+ shrink to ~150px and drop to min-width:820. Each diagram needs its own marker id (f1) — never reuse across SVGs on the page.
ladderlevels that build on each other (maturity, tiers)
Tweak: each step rises 54px and shifts right 160px. Color ramps coral→amber→indigo→green (cold→warm progress). Add an L4 by continuing the pattern (raise viewBox top). Risers are thin connectors, not arrows. In production on the software-factory page.
matrix-2x2two axes of trade-off; plotting options
Tweak: quadrant tints at 0.45–0.6 opacity; solid axes on the left/bottom, dashed crosshair through the middle. Plot dots with --*-solid + white labels (never var(--green) — that flips light in dark mode). If labels would collide, number the dots and legend them in HTML below. Keep dots out of the caption rows.
branchone start → two diverging paths → two outcomes
Tweak: coral = the worse path, teal = the better. Outcome boxes are the only solid-filled ones → --*-solid + white text. Three marker ids (br0/1/2), one per arrow color. In production on the what-matters essay (被動 vs 主動).
convergeN inputs combine into one result (not a trade-off)
Tweak: N input nodes in a row feeding one outcome via curved arrows that converge near the outcome's top-center. Outcome is the solid box (--green-solid + white). For 2 or 4 inputs, re-space evenly and re-aim the arrow control points at the same convergence point. In production on the what-matters triad.
panelsN parallel options/cases compared side by side
Tweak: 3 equal panels (288 wide, 6px gaps) tinted with *-bg. The little schematics inside (one deep bar / shallow stacked bars / several deep bars) carry the metaphor. For 2 panels widen to ~445; the panels use tints + accent text, so no -solid needed. In production on the what-matters essay (三種找答案).
